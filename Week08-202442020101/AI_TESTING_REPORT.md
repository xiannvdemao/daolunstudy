## 二、遇到的问题和解决方法

### 2.1 环境配置问题

**问题**: 系统权限限制导致无法安装 gcc 编译器

**解决方法**:
1. 确认系统并非处于只读模式
2. 使用系统包管理器安装必要依赖
3. 配置 Rust 工具链环境变量

### 2.2 测试用例冲突

**问题**: 生成的测试用例与已有测试重复定义

**解决方法**:
1. 检查并删除重复的测试函数
2. 统一测试命名规范
3. 在生成前进行代码扫描避免重复

### 2.3 代码覆盖率工具兼容性

**问题**: 不同覆盖率工具（tarpaulin, llvm-cov, grcov）输出格式不一致

**解决方法**:
1. 选择 llvm-cov 作为统一工具
2. 配置工作空间级别的覆盖率报告
3. 生成标准化的 HTML 和 JSON 报告

### 2.4 Crate 依赖管理

**问题**: 分离 lexer 和 parser 后出现循环依赖和模块引用问题

**解决方法**:
1. 在 Cargo.toml 中配置工作空间
2. 建立清晰的 crate 依赖关系
3. 在主项目中重新导出子 crate 的 API 保持兼容性

### 2.5 测试迁移挑战

**问题**: 将测试从原有模块迁移到新 crate 时出现覆盖率下降

**解决方法**:
1. 逐步迁移测试用例
2. 针对新结构补充缺失的测试
3. 持续监控覆盖率变化

### 2.6 覆盖率对比分析

**问题**: 需要清晰展示重构前后的覆盖率变化

**解决方法**:
1. 生成详细的覆盖率对比报告
2. 分析各模块变化原因
3. 提出针对性改进建议
